# Windows 11 ネイティブで動かすモダンなTODOアプリケーション

WinUI を使用して、Windows 11 に自然になじむモダンな見た目のTODOアプリを作成してください。

## 要件

- 画面は WinUI 3 を前提に設計すること。
- Windows 11 の標準アプリらしい、洗練された見た目にすること。
- ダークモードとライトモードのどちらでも見やすい配色にすること。
- 余白、角丸、影、アクセントカラーを適切に使い、古い業務アプリ風の見た目にしないこと。
- 画面幅の変化にある程度対応できるレイアウトにすること。

## アプリ機能

- TODO の追加
- TODO の完了切り替え
- TODO の削除
- TODO の編集
- 完了済みと未完了の区別表示
- フィルタ表示
  - すべて
  - 未完了
  - 完了済み

## 画面構成

- 上部にアプリ名と簡単な説明を表示すること。
- 中央に TODO の入力欄と追加ボタンを配置すること。
- その下に TODO 一覧を表示すること。
- 一覧の各項目には、チェックボックス、タイトル、削除ボタン、編集ボタンを用意すること。
- 空の状態では、入力を促す分かりやすいメッセージを表示すること。

## デザイン方針

- 全体の印象は明るく、余白を広めに取ること。
- 文字サイズは読みやすさを優先し、階層がわかるようにすること。
- ボタンや入力欄は統一感のあるデザインにすること。
- 単純な白背景だけにせず、アクセントを加えて少し個性のあるUIにすること。
- アイコンは必要に応じて使用し、視認性を上げること。

## 実装方針

- MVVM を意識した構成にすること。
- TODO のデータはモデルとして分けること。
- 画面の状態管理は ViewModel 側にまとめること。
- コードはできるだけ読みやすく、責務が分かれるように整理すること。

## 期待する完成イメージ

- 起動した瞬間に使い方が分かること。
- 操作が少なく、すぐにTODOを登録・整理できること。
- 見た目が Windows 11 の標準アプリに近い完成度であること。
