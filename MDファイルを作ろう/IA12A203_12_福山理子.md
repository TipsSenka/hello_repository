# アプリケーション開発指示書

## プロジェクト概要

### アプリケーション名
ToDo Timer

### 概要
シンプルなToDoリストとポモドーロタイマーを組み合わせたWebアプリケーションを作成する。

HTML・CSS・JavaScriptのみで実装し、サーバーやデータベースは使用しない。

---

# 開発ルール

- HTML・CSS・JavaScriptのみで実装する
- 外部ライブラリは使用しない
- ES6以降のJavaScriptを使用する
- コードには適度にコメントを書く
- 可読性を重視する
- ファイル構成は以下とする

```
/
├── index.html
├── style.css
└── script.js
```

---

# 必須機能

## 1. ToDo管理

### タスク追加

- テキスト入力欄
- 追加ボタン
- Enterキーでも追加可能

### タスク一覧

各タスクには以下を表示する

- タスク名
- 完了チェックボックス
- 削除ボタン

### タスク完了

- チェックすると取り消し線を表示
- 完了状態を切り替え可能

### タスク削除

削除ボタンを押すと一覧から削除する

---

## 2. ポモドーロタイマー

### 基本仕様

- 初期値25分
- スタート
- 一時停止
- リセット

### タイマー表示

以下の形式で表示

```
25:00
```

### タイマー終了

- アラート表示
- タイトルを
「時間になりました」
に変更する

---

## 3. データ保存

ブラウザのLocalStorageを使用する

保存する内容

- タスク一覧
- タスク完了状態

ページを再読み込みしても保持されること

---

# JavaScript仕様

以下のように関数を分けること

```
addTask()

deleteTask()

toggleTask()

saveTasks()

loadTasks()

startTimer()

pauseTimer()

resetTimer()

updateTimerDisplay()
```

イベントリスナーは最後にまとめて記述すること

---

# エラーハンドリング

以下に対応する

- 空文字では追加できない
- タイマーが動作中は二重起動しない
- LocalStorageにデータが存在しない場合でもエラーにならない

---

# コーディング規約

変数名・関数名は英語

例

```
taskList
taskInput
remainingTime
timerInterval
```

定数はconstを使用

再代入するもののみletを使用

varは禁止

---

# 完成条件

以下を満たしたら完成

- 全機能が正常に動作する
- コンソールエラーが発生しない
- リロード後もToDoが保持される
- レスポンシブ表示が崩れない
- HTML・CSS・JavaScriptのみで動作する

---

# AIへの指示

以下の順番で開発すること

1. HTML作成
2. CSS作成
3. JavaScript作成
4. 動作確認
5. 必要に応じてリファクタリング

各工程ごとに完成したコードを提示し、仕様を満たしているか自己確認を行うこと。
