# JavaScriptピアノ制作 指示書

## 1. 概要
ブラウザ上で動作する、Web Audio APIを使ったシンプルな仮想ピアノを作成する。
マウス操作とPCキーボード操作の両方で演奏できるようにする。

## 2. 使用技術
- HTML / CSS / JavaScript(素のJS、フレームワーク不要)
- 音の生成には `Web Audio API`(`AudioContext`, `OscillatorNode`, `GainNode`)を使用する
- 外部ライブラリ・音源ファイルは使用しない(オシレーターで音を合成する)

## 3. 機能要件

### 3.1 鍵盤
- 1オクターブ+2音程度(例: C4〜D5)を表示する
- 白鍵と黒鍵を見た目で正しく区別し、黒鍵は白鍵の間の正しい位置に重ねて配置する
- 各鍵盤に対応するPCキーボードのキーを表示する(例: 「A」「S」「D」など)

### 3.2 音を鳴らす仕組み
- 鍵盤ごとに周波数(Hz)をあらかじめ定義しておく
- `OscillatorNode`を生成し、対応する周波数をセットして再生する
- `GainNode`を挟み、音量を急に0から立ち上げる(アタック)ことでノイズ(クリック音)を防ぐ
- キーを離したら`GainNode`の音量を滑らかに0へ下げてから`osc.stop()`する(リリース)

### 3.3 操作方法
- マウス/タップ: `mousedown`/`mouseup`(スマホ対応なら`touchstart`/`touchend`も)で発音・停止
- キーボード: `keydown`/`keyup`イベントで発音・停止。長押しで音が重複して鳴らないよう、押下中のキーを管理する(Setなどで押しっぱなし判定)
- 鍵盤を押している間は色を変える等、視覚的なフィードバックを入れる

### 3.4 音色(任意)
- `oscillator.type`を切り替えるセレクトボックスを用意する
  - `sine`(サイン波・丸い音)
  - `triangle`(三角波・柔らかい音)
  - `sawtooth`(のこぎり波・明るい音)
  - `square`(矩形波・レトロな音)

## 4. 非機能要件
- `AudioContext`はブラウザの自動再生制限があるため、ユーザーの最初の操作(クリック/キー入力)時に`resume()`する
- PCのみでなくスマホでも操作できるようにタッチイベントに対応する
- コードは1ファイル(HTML)にまとめても、HTML/CSS/JSを分割してもよい

## 5. 発展要素(余裕があれば)
- オクターブ切り替えボタン
- 和音(コード)を鳴らすプリセットボタン
- 録音・再生機能(演奏した音符と時間を配列に記録し再生)
- サステインペダル(離しても音を伸ばす)のON/OFF
- ADSR(アタック・ディケイ・サステイン・リリース)を細かく調整するスライダー
