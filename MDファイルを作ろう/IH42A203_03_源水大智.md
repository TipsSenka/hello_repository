# プロジェクト計画: Blenderによる扇子STL動的生成

- 対象Blenderバージョン: 5.1.2（Python API `bpy` 前提）
- 実行方式: ヘッドレスCLIスクリプト（`blender --background --python script.py`）
  - `bpy.types.Operator`（`bl_options={'REGISTER', 'UNDO'}`）として実装し、Blender GUI上では「最後の操作を調整」パネル（F6/左下パネル）で数値を再計算しながら調整可能にする
  - CLI実行時はデフォルト値／引数がOperatorプロパティの初期値になる

## コンセプト

- 紙(扇面)は使わず、**PLA(3Dプリント樹脂)のみ**で全パーツを構成する
- 骨(リブ)を要(かなめ)のピンで連結し、実際に**開閉できる可動モデル**にする
- 各骨には短冊状の台形パネルを一体成形し、開いた時に隣接パネルとオーバーラップさせることで扇面のように連続して見えるようにする

## 構造・パーツ構成

1. **仲骨（なかぼね）**: 中央部の骨。骨本体＋短冊パネルを一体成形。全て同一形状。
2. **親骨（おやぼね）**: 両端2本。仲骨より厚みを持たせ、閉じた際に内側の仲骨を保護する。
3. **要（かなめ）**: 各骨の根本にピン穴を設け、軸ピンを通して回転可能にする。
   - ピン自体は生成しない（穴のみモデル化）。市販の丸棒・竹串などをユーザーが別途用意する前提。

## パラメータ（デフォルト値）

| パラメータ | デフォルト値 | 備考 |
|---|---|---|
| 骨の本数 `n` | 10 | 親骨2本＋仲骨8本 |
| 開き角度 `θ_open` | 180° | |
| 半径(扇の長さ) `radius` | 200mm | |
| 仲骨の厚み | 2mm | |
| 親骨の厚み | 4mm | `guard_rib_thickness_multiplier = 2.0` |
| 短冊オーバーラップ率 `overlap_ratio` | 0.2（20%重ね） | 隣接パネルとの重なり量 |
| 要のピン穴径 `pin_hole_diameter` | 2.0mm | 竹串/丸棒想定 |

短冊パネルの上底・下底の長さは上記パラメータ（半径・骨1本あたりの角度・オーバーラップ率）から計算式で自動算出する。Operatorパネル上で数値を直接上書きすることも可能にする。

## 開閉時の干渉確認

- コード側での自動干渉チェック（assert等）は行わない
- **Blender上で実際に骨を開閉角度分回転させ、目視で確認する**運用とする

## STL出力

- 出力形式: **個別STLファイル**（可動モデルのため一体出力はできない）
- 同一形状のパーツは1種類のみ出力し、スライサー側で必要数を複製する
  - `rib_middle.stl`（仲骨、1個分の形状）
  - `rib_guard.stl`（親骨、1個分の形状）
- 実際に印刷する数量: 仲骨 `n-2` 個、親骨 2個
