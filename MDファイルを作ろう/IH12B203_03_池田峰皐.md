# Google Maps API を使った観光案内Webページの要件整理

このファイルには、Google Maps API を使って観光案内の Web ページを作るときに必要な情報をまとめます。
そのまま実装依頼のプロンプトとしても使えるように、目的、必要機能、データ設計、設定手順を整理しています。

## 目的

- 観光客が地図上でスポットを見つけやすいようにする
- スポットの詳細情報を一覧と地図の両方から確認できるようにする
- 地図、一覧、詳細表示を連動させて、操作を分かりやすくする
- スマートフォンでも見やすく、触りやすい画面にする

## 作りたい画面

- ヘッダー
	- サイト名
	- 簡単な説明文
	- 検索欄
	- カテゴリ絞り込みボタン
- メインエリア
	- Google マップ
	- スポットのマーカー
- サイドバーまたは下部エリア
	- 観光スポット一覧
	- 選択中スポットの詳細カード
- フッター
	- 注意事項
	- API 利用に関する補足

## 必須機能

1. Google Maps API を使って地図を表示する。
2. 観光スポットを複数表示し、それぞれにマーカーを置く。
3. マーカーをクリックすると、スポット名、説明、営業時間、アクセス情報などを表示する。
4. 一覧からスポットを選ぶと、地図がその場所へ移動する。
5. 検索機能または絞り込み機能を入れる。
6. レスポンシブ対応にする。
7. 実在の観光地、またはサンプルデータのどちらでもよい。

## あると便利な機能

- 現在地取得と、現在地周辺のスポット表示
- カテゴリ別フィルタリング
	- 飲食
	- 歴史
	- 自然
	- 買い物
- InfoWindow か詳細カードの表示
- スマホで押しやすいボタン設計
- 一覧の並び替え
	- 人気順
	- 距離順
	- カテゴリ順

## データに含めたい項目

スポットごとに、少なくとも次の情報を持たせると使いやすいです。

- id
- name
- description
- category
- latitude
- longitude
- address
- hours
- access
- imageUrl
- websiteUrl
- phone

## データ構造の例

```json
{
	"id": 1,
	"name": "サンプル観光地",
	"description": "景色がきれいな人気スポットです。",
	"category": "自然",
	"latitude": 35.681236,
	"longitude": 139.767125,
	"address": "東京都千代田区",
	"hours": "9:00 - 17:00",
	"access": "駅から徒歩10分",
	"imageUrl": "/images/sample.jpg",
	"websiteUrl": "https://example.com",
	"phone": "03-0000-0000"
}
```

## Google Maps API で必要な設定

- Google Cloud Console でプロジェクトを作成する
- Maps JavaScript API を有効化する
- 必要に応じて Places API も有効化する
- API キーを発行する
- API キーは環境変数で管理する
- API キーをソースコードに直書きしない
- 利用制限のため、HTTP リファラー制限を設定する

## 実装の流れ

1. ページ読み込み時に地図を初期化する。
2. スポット一覧をデータから生成する。
3. 各スポットに対応するマーカーを作成する。
4. マーカークリック時に詳細情報を表示する。
5. 一覧クリック時に地図を対象地点へ移動する。
6. 検索やカテゴリ絞り込みで表示対象を切り替える。
7. モバイル表示時のレイアウトを整える。

## 推奨ファイル構成

- index.html
- style.css
- script.js
- data/spots.json
- .env または環境変数設定ファイル

## デザイン方針

- 旅の雰囲気が伝わる、明るく親しみやすい見た目にする
- 地図が主役になるように、色数は増やしすぎない
- 文字サイズと余白を十分に取る
- モバイルでは 1 カラム、PC では 2 カラム以上にする

## 実装時の注意点

- 初心者にも分かりやすいコードにする
- コメントは最小限にする
- 将来スポット情報を増やしやすい構造にする
- 画像やアイコンを使うなら、扱いやすいものを選ぶ

## 出力してほしいもの

1. 完成した Web ページのコード
2. 必要なファイル構成
3. Google Maps API の設定手順
4. 実行方法
5. 改善案があれば提案
