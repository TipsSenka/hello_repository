# JavaScriptで電卓を作る — 作業計画

## 概要
- 目的: ブラウザ上で動作するシンプルで拡張しやすい電卓を`HTML/CSS/JavaScript`で実装する。
- 想定対象: 初級〜中級の学習者、授業や課題用のサンプルプロジェクト。

## 成果物
- `index.html` — 電卓UI
- `styles.css` — 見た目（レスポンシブ対応）
- `script.js` — 入力処理と計算ロジック（テスト可能な純粋関数で評価）
- `README.md` — 実行方法・仕様・拡張案

## 前提条件
- ブラウザ（最新のChrome/Edge/Firefoxなど）で動作すること。
- セキュリティのため`eval`は使わない（シャントイングヤード等で式を評価）。

## スコープ（含むもの／含まないもの）
- 含む: 四則演算、小数、符号反転、パーセント、クリア、バックスペース、キーボード入力対応
- 含まない（初期）: 科学関数、履歴の永続化（拡張で対応）

## 詳細作業計画（ステップ）

1. 要件定義（完了）
	- 基本機能と非機能要件の確認（アクセシビリティ、レスポンシブ、精度）

2. UI設計（ワイヤーフレーム）
	- ディスプレイ領域（式表示 + 結果表示）を上部に配置
	- ボタンは4列グリッド（数字、演算子、機能ボタン）
	- 各ボタンに`data-key`属性と`aria-label`を付与
	- ドラフトを1枚（PNGまたはASCIIワイヤー）で作成

3. HTML作成
	- `div.calculator` 内に `div.display` と `div.keys` を用意
	- 各ボタンに役割を示す`data-key`（例: `data-key="7"`, `data-key="add"`, `data-key="clear"`）を付ける

4. CSS作成
	- CSS Gridでボタンを配置（`grid-template-columns: repeat(4, 1fr);`）
	- モバイル優先でフォントとボタンサイズを調整
	- :focus / :active スタイルとカラーコントラスト確認

5. JavaScript実装（コアロジック）
	- 状態管理変数を定義（例: `currentInput`, `operator`, `operandStack`）
	- 入力ハンドラ: `appendDigit()`, `applyOperator()`, `calculate()`, `clear()`, `backspace()`
	- キーボード対応: `keydown` イベントでボタン操作を再利用
	- 評価エンジン: トークン化 → シャントイングヤードでRPN生成 → RPNを評価
	- 浮動小数点対策: 表示は最大12桁等で丸め（内部は必要ならライブラリを検討）

6. テスト（ユニット + 手動）
	- 単体: 評価関数に対するユニットテスト（例: `1+2*3 = 7`）
	- 手動: UIでの入力、キーボード、エラー表示（0除算など）を確認
	- テストケース一覧を`tests/`または`README.md`に記載

7. デバッグとエッジケース対応
	- 連続演算子の扱い、先頭ゼロ、小数点の重複、空入力での`=`押下などを修正
	- エラーメッセージの表示（`Error`等）

8. ドキュメント作成
	- `README.md` にビルド/実行手順、キーボード操作一覧、既知の制限を明記

9. 拡張案（優先順）
	- 履歴パネル（再利用可能）
	- メモリ機能（M+, M-, MR, MC）
	- 科学関数モード（sin, cos, log 等）
	- テーマ切替（ライト/ダーク）

## スケジュール案（目安）
- 1日目: ワイヤーフレーム + `index.html` の骨組み
- 2日目: `styles.css` とボタンレイアウト調整
- 3日目: `script.js` の基本入力処理とキーボード対応
- 4日目: 評価ロジック（シャントイングヤード）実装とユニットテスト
- 5日目: デバッグ、アクセシビリティ調整、README作成

## 備考（実装上の注意）
- セキュリティのために`eval`を使わないこと
- 表示はユーザーにわかりやすいように丸め・桁区切りを検討
- 国際化の小数記号（`,` vs `.`）は将来的に対応可能
