# 世界時計アプリの作り方

この文書では、JavaScriptで「世界時計アプリ」を作るときの流れと、機能ごとの実装方法、
サンプルコードをまとめます。

## できあがるもの

- 世界中の地域（都市）の中から、好きな地域を最大5つまで選んで登録できる（5つは上限であり、1つや2つだけの登録でもよい）
- 登録した地域の現在時刻を一覧で表示する（1秒ごとに更新）
- それぞれの地域について、日本時間との時差を表示する
- ウィンドウを最小化したとき専用の、コンパクトな表示に切り替わる
- （発展）その他のおもしろ機能（後述）

## 用意するファイル

- `index.html`
- `style.css`
- `script.js`

## 作り方の流れ

1. 画面の土台（一覧表示エリア・地域選択UI）を `index.html` で作る
2. カードやリストの見た目を `style.css` で整える
3. タイムゾーンごとの時刻計算を `script.js` で作る
4. 地域を選択する機能を作る（追加・削除）
5. 1秒ごとに時刻を更新する仕組みを作る
6. 日本時間との時差を計算して表示する
7. 選択した地域を `localStorage` に保存し、次回起動時も復元する
8. 最小化時のコンパクト表示を作る
9. 余裕があれば追加機能を実装する

## 開発ロードマップ

一度に全部を作ろうとせず、下のフェーズ順に「動くもの」を少しずつ育てていくと
挫折しにくいです。各フェーズの最後に「完了の目安」をチェックしながら進めましょう。

### フェーズ0：準備

- `index.html` / `style.css` / `script.js` の3ファイルを作成する
- 都市データ（都市名・タイムゾーンID）の配列を用意する
- **完了の目安**：ブラウザで `index.html` を開き、真っ白でもエラーなく表示される

### フェーズ1：MVP（最小限の動く時計）

- 東京など、地域1つだけを画面に表示する
- `setInterval` で1秒ごとに時刻を更新する（機能2）
- **完了の目安**：画面の時計が1秒ずつ正しく進む

### フェーズ2：複数地域の登録・一覧表示

- 地域選択UI（`<select>` など）から地域を追加できるようにする（機能1）
- 追加した地域をカードとして一覧表示し、削除もできるようにする
- 登録数の上限（最大5つ）を超えたら追加ボタンを無効化する
- **完了の目安**：2〜5個の地域を自由に追加・削除しても表示が崩れない

### フェーズ3：時差表示

- 日本時間との時差を計算する関数を作る（機能3）
- 各カードに「日本との時差 +◯時間」を表示する
- **完了の目安**：サマータイム中の地域（ロンドン・ニューヨークなど）でも
  時差が正しく表示される

### フェーズ4：保存機能

- 選択した地域を `localStorage` に保存する（機能5）
- ページを開いたときに前回の登録内容を復元する
- **完了の目安**：ブラウザを閉じて開き直しても、登録した地域が消えない

### フェーズ5：最小化時の専用表示

- タブが非アクティブなときにタイトルを書き換える（パターンA）
- 「コンパクト表示」ボタンでミニ表示に切り替えられるようにする（パターンB）
- **完了の目安**：タブを切り替えたときとコンパクト表示ボタンを押したとき、
  それぞれ想定通りの表示になる

### フェーズ6：見た目の仕上げ

- カードのデザイン、フォント、配色、余白などを `style.css` で整える
- スマートフォンなど画面が狭い環境でも見やすいようにレイアウトを調整する
- **完了の目安**：友達や家族に見せて「使い方が分かる」と言ってもらえる

### フェーズ7：発展機能（余裕があれば・優先度順）

余裕がある場合は、実装のしやすさ・使う頻度を考えて次の順番で挑戦するのがおすすめです。

1. **タイムゾーン変換ツール**（時差計算の応用なので着手しやすい）
2. **会議に適した時間帯の提案**（変換ツールの考え方をそのまま流用できる）
3. **アラーム・通知機能**（Notification APIの許可周りがやや難しい）
4. **世界地図との連携**（座標変換や見た目の調整に時間がかかるため最後に）

- **完了の目安**：それぞれの機能を1つ作るごとに単体で動作確認してから
  次の機能に進む（一度に複数を作ろうとしない）

## 機能1：好きな地域を最大5つまで選んで一覧表示する

### 考え方

- あらかじめ「都市名」と「タイムゾーンID（IANA名）」の組を配列で用意しておく
  - 例：`{ city: 'ロンドン', timeZone: 'Europe/London' }`
- タイムゾーンIDは [IANA Time Zone Database](https://www.iana.org/time-zones) の名称
  （`Asia/Tokyo`、`America/New_York` など）を使う
- ユーザーは `<select>` やチェックボックスの一覧から地域を選び、
  「追加」ボタンで登録リストに加える
- 登録できる上限（例：5つ）を決めておき、超えたら追加ボタンを無効化する
- 登録済みの地域はカード（またはリストの行）として画面に並べて表示する
- 各カードに「削除」ボタンを付け、いつでも入れ替えられるようにする

### 都市データの例

```javascript
const CITY_LIST = [

  { city: 'ニューヨーク', timeZone: 'America/New_York' },
  { city: 'ロサンゼルス', timeZone: 'America/Los_Angeles' },
  { city: 'ロンドン',     timeZone: 'Europe/London' },
  { city: 'パリ',        timeZone: 'Europe/Paris' },
  { city: 'モスクワ',     timeZone: 'Europe/Moscow' },
  { city: 'ドバイ',       timeZone: 'Asia/Dubai' },
  { city: 'ニューデリー', timeZone: 'Asia/Kolkata' },
  { city: 'シンガポール', timeZone: 'Asia/Singapore' },
  { city: '香港',        timeZone: 'Asia/Hong_Kong' },
  { city: 'シドニー',     timeZone: 'Australia/Sydney' },
  { city: 'ホノルル',     timeZone: 'Pacific/Honolulu' },
];

const MAX_SELECTED = 5;
let selectedCities = [];
```

### 追加・削除の処理

```javascript
function addCity(timeZone) {
  if (selectedCities.length >= MAX_SELECTED) {
    alert(`登録できる地域は${MAX_SELECTED}つまでです`);
    return;
  }
  if (selectedCities.some((item) => item.timeZone === timeZone)) {
    return; // 重複追加を防ぐ
  }
  const city = CITY_LIST.find((item) => item.timeZone === timeZone);
  selectedCities.push(city);
  saveSelectedCities();
  renderClockList();
}

function removeCity(timeZone) {
  selectedCities = selectedCities.filter((item) => item.timeZone !== timeZone);
  saveSelectedCities();
  renderClockList();
}
```

## 機能2：現在時刻をリアルタイムで表示する

### 考え方

- `Intl.DateTimeFormat` にタイムゾーンを指定すると、そのタイムゾーンの時刻を
  簡単に取得できる（自前で時差計算をしなくてよい）
- `setInterval` を使い、1秒ごとに全地域の表示を更新する
- 日付・曜日・昼/夜のアイコンなども合わせて表示すると分かりやすい

```javascript
function formatTime(timeZone) {
  return new Intl.DateTimeFormat('ja-JP', {
    timeZone,
    hour: '2-digit',
    minute: '2-digit',
    second: '2-digit',
    hour12: false,
  }).format(new Date());
}

function formatDate(timeZone) {
  return new Intl.DateTimeFormat('ja-JP', {
    timeZone,
    year: 'numeric',
    month: 'short',
    day: 'numeric',
    weekday: 'short',
  }).format(new Date());
}

function isDaytime(timeZone) {
  const hour = Number(
    new Intl.DateTimeFormat('en-US', { timeZone, hour: 'numeric', hour12: false }).format(new Date())
  );
  return hour >= 6 && hour < 18;
}
```

### 一覧の描画と更新ループ

```javascript
const clockList = document.getElementById('clockList');

function renderClockList() {
  clockList.innerHTML = '';
  selectedCities.forEach((item) => {
    const card = document.createElement('li');
    card.className = 'clock-card';
    card.dataset.timeZone = item.timeZone;
    card.innerHTML = `
      <div class="clock-card__header">
        <span class="clock-card__city">${item.city}</span>
        <span class="clock-card__icon">${isDaytime(item.timeZone) ? '☀️' : '🌙'}</span>
        <button class="clock-card__remove" aria-label="削除">×</button>
      </div>
      <div class="clock-card__time">${formatTime(item.timeZone)}</div>
      <div class="clock-card__date">${formatDate(item.timeZone)}</div>
      <div class="clock-card__diff">${formatDiffLabel(item.timeZone)}</div>
    `;
    card.querySelector('.clock-card__remove')
      .addEventListener('click', () => removeCity(item.timeZone));
    clockList.appendChild(card);
  });
}

function tick() {
  selectedCities.forEach((item) => {
    const card = clockList.querySelector(`[data-time-zone="${item.timeZone}"]`);
    if (!card) return;
    card.querySelector('.clock-card__time').textContent = formatTime(item.timeZone);
    card.querySelector('.clock-card__icon').textContent = isDaytime(item.timeZone) ? '☀️' : '🌙';
  });
  updateMiniDisplay();
}

setInterval(tick, 1000);
```

## 機能3：日本時間との時差を表示する

### 考え方

- タイムゾーンをまたいだ時差は、サマータイム（夏時間）の影響で単純な引き算では
  ずれることがあるため、「同じ瞬間の時刻を2つのタイムゾーンで文字列化し、
  差分を計算する」方法が確実
- `Intl.DateTimeFormat` の `hour` / `minute` を数値として取り出し、
  UTCからのオフセット（分単位）を比較する

```javascript
function getUtcOffsetMinutes(timeZone, date = new Date()) {
  const formatter = new Intl.DateTimeFormat('en-US', {
    timeZone,
    hour12: false,
    year: 'numeric', month: '2-digit', day: '2-digit',
    hour: '2-digit', minute: '2-digit', second: '2-digit',
  });
  const parts = formatter.formatToParts(date).reduce((acc, part) => {
    acc[part.type] = part.value;
    return acc;
  }, {});
  const asUTC = Date.UTC(
    Number(parts.year), Number(parts.month) - 1, Number(parts.day),
    Number(parts.hour), Number(parts.minute), Number(parts.second)
  );
  return (asUTC - date.getTime()) / 60000;
}

function formatDiffLabel(timeZone) {
  const tokyoOffset = getUtcOffsetMinutes('Asia/Tokyo');
  const targetOffset = getUtcOffsetMinutes(timeZone);
  const diffMinutes = targetOffset - tokyoOffset;

  if (diffMinutes === 0) {
    return '日本と同じ時刻';
  }

  const sign = diffMinutes > 0 ? '+' : '-';
  const abs = Math.abs(diffMinutes);
  const hours = Math.floor(abs / 60);
  const minutes = abs % 60;
  const minutesText = minutes > 0 ? `${minutes}分` : '';
  return `日本との時差 ${sign}${hours}時間${minutesText}`;
}
```

### ポイント

- `Asia/Tokyo` は常に UTC+9（サマータイムなし）なので基準にしやすい
- 分単位まで計算しておくと、インドなど「+5時間30分」のような
  30分刻みの時差にも対応できる

## 機能4：最小化したときの専用表示（ミニ表示）

「最小化」は環境によって意味が変わるため、下の2パターンを実装しておくと安心です。

### パターンA：ブラウザタブが非アクティブ（バックグラウンド）になったとき

- `document.addEventListener('visibilitychange', ...)` で検知する
- タブがバックグラウンドのときは `document.title` を書き換えて、
  タブを見るだけで代表都市の時刻が分かるようにする

```javascript
const representativeTimeZone = 'Asia/Tokyo'; // 一覧の先頭など、代表の地域

function updateMiniDisplay() {
  if (document.hidden) {
    document.title = `${formatTime(representativeTimeZone)}｜世界時計`;
  } else {
    document.title = '世界時計アプリ';
  }
}

document.addEventListener('visibilitychange', updateMiniDisplay);
```

### パターンB：アプリ内の「コンパクトモード」ボタン

- ウィンドウが小さくなっても崩れない、専用の省スペースレイアウトを用意する
- CSSクラスの切り替えで「通常表示」と「コンパクト表示」を切り替える
- コンパクト表示では、都市名を省略した略称（例：TYO / NYC / LON）と時刻だけを
  横一列で表示する

```html
<button id="compactToggle">コンパクト表示に切り替え</button>
<div id="clockContainer" class="clock-container">
  <!-- 通常表示のカード一覧 -->
</div>
<div id="miniBar" class="mini-bar is-hidden">
  <!-- コンパクト表示：TYO 21:45 / NYC 08:45 のように横並び -->
</div>
```

```css
.mini-bar {
  display: flex;
  gap: 12px;
  font-size: 14px;
  padding: 4px 8px;
}
.mini-bar.is-hidden,
.clock-container.is-hidden {
  display: none;
}
```

```javascript
const compactToggle = document.getElementById('compactToggle');
const clockContainer = document.getElementById('clockContainer');
const miniBar = document.getElementById('miniBar');
```

- Electronなどでデスクトップアプリ化する場合は、OSの「最小化」イベントを
  トレイアイコンや小さいウィジェットウィンドウに反映させることもできる
  （その場合は `BrowserWindow` の `minimize` イベントを使う）

## 使い方

1. `index.html` をブラウザで開く
2. 地域一覧から好きな地域を選び、「追加」ボタンで登録する（最大5つ）
3. 登録した地域が一覧に並び、時刻・日付・日本時間との時差が表示される
4. 「コンパクト表示に切り替え」ボタンでミニ表示に切り替えられる
5. タブを別のウィンドウに切り替えると、タブのタイトルに時刻が表示される
6. 選んだ地域はブラウザを閉じても保存される

### 1. 世界地図との連携

```javascript
}

function renderMapPins() {
```

### 2. タイムゾーン変換ツール

- 「基準地域」「基準時刻」を入力すると、登録済みの全地域の対応時刻を
  計算して一覧表示する機能
- 考え方：入力された時刻を「基準地域のタイムゾーンでのその時刻」として
  UTC時刻に変換し、そこから各地域の時刻に変換し直す

```html
<div class="converter">
  <select id="baseTimeZone"><!-- 地域一覧 --></select>
  <input type="time" id="baseTime" value="15:00">
  <input type="date" id="baseDate">
  <button id="convertButton">変換する</button>
</div>
<ul id="convertResult"></ul>
```

```javascript
function convertTime(baseTimeZone, dateString, timeString) {
  // 「基準タイムゾーンでの dateString + timeString」をUTCミリ秒に変換する
  const [hour, minute] = timeString.split(':').map(Number);
  const naiveUTC = new Date(`${dateString}T${timeString}:00Z`).getTime();
  const baseOffsetMinutes = getUtcOffsetMinutes(baseTimeZone, new Date(naiveUTC));
  const trueUTC = naiveUTC - baseOffsetMinutes * 60000;
  return new Date(trueUTC);
}

function showConvertedTimes() {
  const baseTimeZone = document.getElementById('baseTimeZone').value;
  const dateString = document.getElementById('baseDate').value;
  const timeString = document.getElementById('baseTime').value;
  const targetInstant = convertTime(baseTimeZone, dateString, timeString);

  const resultList = document.getElementById('convertResult');
  resultList.innerHTML = selectedCities.map((item) => {
    const time = new Intl.DateTimeFormat('ja-JP', {
      timeZone: item.timeZone, hour: '2-digit', minute: '2-digit', hour12: false,
    }).format(targetInstant);
    return `<li>${item.city}：${time}</li>`;
  }).join('');
}

document.getElementById('convertButton').addEventListener('click', showConvertedTimes);
```

- `getUtcOffsetMinutes()` は前述（機能3・時差計算）で作った関数をそのまま使い回せる

### 3. 会議に適した時間帯の提案

- 登録した全地域について、それぞれの現地時間が「9:00〜21:00」に収まる
  UTC上の時間帯（時間帯の候補）を1時間刻みで総当たりし、条件を満たす
  時間帯だけを一覧表示する

```javascript
const WORK_START_HOUR = 9;
const WORK_END_HOUR = 21;

function findGoodMeetingHours(dateString) {
  const candidates = [];

  for (let utcHour = 0; utcHour < 24; utcHour += 1) {
    const candidateUTC = new Date(`${dateString}T${String(utcHour).padStart(2, '0')}:00:00Z`);

    const fitsAll = selectedCities.every((item) => {
      const localHour = Number(
        new Intl.DateTimeFormat('en-US', {
          timeZone: item.timeZone, hour: 'numeric', hour12: false,
        }).format(candidateUTC)
      );
      return localHour >= WORK_START_HOUR && localHour < WORK_END_HOUR;
    });

    if (fitsAll) {
      candidates.push(candidateUTC);
    }
  }

  return candidates;
}

function renderMeetingSuggestions(dateString) {
  const goodHours = findGoodMeetingHours(dateString);
  const suggestionList = document.getElementById('meetingSuggestions');

  if (goodHours.length === 0) {
    suggestionList.innerHTML = '<li>全員の勤務時間に収まる時間帯が見つかりませんでした</li>';
    return;
  }

  suggestionList.innerHTML = goodHours.map((utcTime) => {
    const times = selectedCities.map((item) => {
      const time = new Intl.DateTimeFormat('ja-JP', {
        timeZone: item.timeZone, hour: '2-digit', minute: '2-digit', hour12: false,
      }).format(utcTime);
      return `${item.city} ${time}`;
    }).join(' / ');
    return `<li>${times}</li>`;
  }).join('');
}
```

- 「9:00〜21:00」の範囲はユーザーが自由に変更できるようにすると
  より実用的になる（入力欄を追加してAWORK_START_HOUR/WORK_END_HOURを可変にする）

### 4. アラーム・通知機能

- 「都市」と「その地域での通知時刻」を登録しておき、`tick()` の中で
  現在時刻と一致したらブラウザの通知（Notification API）を出す
- 通知を出すには、事前にユーザーへ許可を求める必要がある

```javascript
const alarms = []; // 例: { timeZone: 'America/New_York', hour: 9, minute: 0, label: 'NY支社の始業' }

function requestNotificationPermission() {
  if (Notification.permission === 'default') {
    Notification.requestPermission();
  }
}

function checkAlarms() {
  alarms.forEach((alarm) => {
    const [hour, minute] = new Intl.DateTimeFormat('en-US', {
      timeZone: alarm.timeZone, hour: 'numeric', minute: 'numeric', hour12: false,
    }).format(new Date()).split(':').map(Number);

    if (hour === alarm.hour && minute === alarm.minute && !alarm.firedToday) {
      new Notification(`${alarm.label}の時間です`, {
        body: `${alarm.timeZone} が ${alarm.hour}:${String(alarm.minute).padStart(2, '0')} になりました`,
      });
      alarm.firedToday = true;
    }

    if (minute !== alarm.minute) {
      alarm.firedToday = false; // 分が変わったら次回また鳴らせるようにリセット
    }
  });
}

requestNotificationPermission();
setInterval(checkAlarms, 1000 * 30); // 30秒ごとに確認
```

- 同じ分に複数回通知が飛ばないよう、`firedToday` のようなフラグで
  「その分にすでに通知したか」を管理する
- 登録したアラームも `localStorage` に保存しておくと、
  ページを開き直しても消えない

## まとめ

世界時計アプリは、①地域データの用意、②タイムゾーンごとの時刻計算、
③一覧表示と更新ループ、④時差計算、⑤最小化時の専用表示、⑥保存機能の
6つがそろうと基本形が完成します。
まずは1地域だけで正しく時刻と時差が表示できることを確認し、
そのあとで複数地域・コンパクト表示・追加機能へと広げていくと進めやすいです。
